How it runs

Schedule. Pull. Dedupe. Deliver.

Five hops, fully observable. If a source fails, the script logs it and retries on the next run - your friend never gets a partial list without knowing why.

1
Schedule
Modal cron fires every Monday morning, kicks off the pull.
2
Chicago
Hits the Chicago Open Data permits API, pulls everything issued in the prior week.
3
Suburbs
Scrapes public permit records from surrounding municipalities, normalised to the same shape.
4
Dedupe
Cross-references against Supabase history so no permit appears twice across weekly runs.
5
Deliver
CSV emailed direct, dropped into a shared Drive folder. One file per week, time-stamped.
